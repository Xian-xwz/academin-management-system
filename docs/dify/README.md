# Dify 知识卡片模板

本目录用于毕业设计 Dify 工作流中的「输入知识点 → 选择知识卡片模板 → 生成生图 prompt」环节。

## 文件说明

| 文件 | 说明 |
|------|------|
| `知识卡片模板.md` | 模板真源文档，包含模板编号、路由分类、示例 prompt 与 JSON 模板信息。 |
| `知识卡片模板节点.py` | Dify 代码节点：输入 `content`，输出单一 `result`（JSON 字符串，内含图片编号、示例、模板）。 |

## 关键参数

- `content`：上游 LLM 全文或 JSON；JSON 时读取 `图片编号` 或 `image_number`；纯文本时从串中提取编号（支持 `1`～`7`、`图片1`、`template_1` 等）。兼容旧版入参 `图片编号` / `image_id`（仅当 `content` 为空时生效）。
- 返回：`result`（String），解析后为 `{ "图片编号", "示例", "模板" }`，其中 `模板` 为 `id` / `name` / `definition` / `style_rules` / `reference_prompt` 对象。

## 维护约定

1. 修改 `知识卡片模板.md` 的示例或 JSON 模板后，如需 Dify 代码节点立即生效，也要同步修改 `知识卡片模板节点.py` 中对应编号的内联模板。
2. 新增图片 8 及后续模板时，需同时更新：
   - `知识卡片模板.md` 顶部「模板编号与分类」表；
   - 新增 `##图片N` 的 `#示例：` 与 `#模板：` JSON 块；
   - `知识卡片模板节点.py` 的 `TEMPLATES` 字典。
3. 所有提示词必须保留「区块内约束」与「无水印 / 无平台角标 / 无 @账号署名」限制，避免生图出现平台 logo、角标或布局标签。
