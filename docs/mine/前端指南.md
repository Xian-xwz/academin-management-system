## 二、前端技术栈（与技术文档一致，落地为 Vue）

- **框架**：Vue 3（`<script setup>` 组合式 API 推荐）。
- **构建**：Vite。
- **路由**：Vue Router 4。
- **状态**：Pinia（用户信息、会话、全局 UI 状态等）。
- **UI 组件库**：Element Plus。
- **HTTP**：Axios；封装统一实例（baseURL、拦截器注入 Token、401 统一处理）。
- **日历 / 课表网格**：FullCalendar（Vue 3 适配包），或与 Element Plus 表格/栅格自研周视图（按团队熟悉度选型）。
- **通用**：UTF-8、中文注释；与后端 RESTful API 对接，统一使用项目约定的请求/响应格式。

## 三、功能模块与前端页面说明

### 1. 用户认证模块

| 项目     | 说明 |
|----------|------|
| 功能说明 | 用户注册、登录、登出；Token 管理；未登录时引导至登录。 |
| 需求依据 | 需求文档 3.3 安全性 — 用户登录认证、权限管理。 |

**1.1 登录页（Login）**

- **路由**（示例）：`/login`。
- **页面要素**：
  - 学号/用户名输入（格式校验）。
  - 密码输入。
  - 「登录」按钮，调用 `POST /api/auth/login`，保存 Token 与用户信息后跳转首页。
  - 跳转「注册」入口。
- **交互**：错误时展示接口返回的 message；支持加载态。

**1.2 注册页（Register）**

- **路由**：`/register`。
- **页面要素**：
  - 学号、用户名、密码、确认密码、邮箱等（与后端接口一致）。
  - 「注册」调用 `POST /api/auth/register`，成功后可跳转登录或直接登录。
- **交互**：前端格式校验 + 后端错误信息展示。

**1.3 全局与认证**

- 请求头统一携带 `Authorization: Bearer {token}`。
- Token 过期或 401 时清除本地登录态并跳转登录页。
- 登录后可选的用户信息展示（头像、学号/昵称）及「退出」入口。

### 3. 学号查询模块

| 项目     | 说明 |
|----------|------|
| 功能说明 | 输入学号后查询毕业要求、选课要求、已选课程、课程分类、还需选修的课程及建议。 |
| 需求依据 | 需求文档 2.1 学号查询功能 FR-001～FR-006。 |
| 接口     | `GET /api/student/{student_id}/academic-info`（技术文档 3.2.1）。 |

**3.1 学号查询页（StudentQuery）**

- **路由**：`/student-query`。
- **页面要素**：
  1. **学号输入区**
     - 输入框 + 「查询」按钮。
     - 前端学号格式校验（与需求 FR-001 一致）；可支持当前登录用户学号自动带入。
  2. **结果展示区**（按接口返回结构渲染）：
     - **毕业要求**（FR-002）：总学分、必修/选修/实践等各类学分要求、其他条件（英语、计算机等）。
     - **选课要求**（FR-003）：当前学期可选课程列表、选课限制、选课时间（若有）。
     - **已选课程**（FR-004）：已选列表、成绩（若有）、学分统计。
     - **课程分类**（FR-005）：按公共必修、专业必修、专业选修、公共选修等分类，每类「已修/应修」学分。
     - **还需选修**（FR-006）：还需的课程类别及学分数、选课建议文案。
- **交互**：查询前 loading；无数据或错误时友好提示；可考虑 Tab 或折叠面板分块展示。

**3.2 组件建议**

- `AcademicInfoCard`：单块信息卡片（毕业要求/已选课程/分类等）。
- `CourseCategoryTable`：课程分类表格。
- `RecommendationList`：选课建议列表。

### 4. AI 问询助手模块

| 项目     | 说明 |
|----------|------|
| 功能说明 | 自然语言问询、多轮对话、触发词调用工作流（如「查询毕业要求」「选课建议」）。 |
| 需求依据 | 需求文档 2.2 AI 问询助手 FR-007～FR-010。 |
| 接口     | `POST /api/ai/chat`、`GET /api/ai/conversations/{conversation_id}`（技术文档 3.3）。 |

**4.1 AI 问询页（AIChat）**

- **路由**：`/ai-chat`。
- **页面要素**：
  1. **对话区域**
     - 消息列表：用户消息、AI 回复；支持多轮对话（FR-009）。
     - 可选展示「知识来源」等（若接口返回 sources）。
  2. **输入区**
     - 文本输入框 + 发送按钮；支持回车发送。
     - 可提供快捷问题（如「查询毕业要求」「选课建议」）一键填入。
  3. **会话**
     - 新建会话、切换会话（若产品需要）；conversation_id 与后端保持一致。
- **交互**：发送后 loading 或「正在输入」态；AI 回复可流式展示（若后端支持 stream）；错误时展示接口错误信息。

**4.2 组件建议**

- `ChatWindow`：聊天窗口容器（消息列表 + 输入框）。
- `MessageBubble`：单条消息气泡（用户/AI）。
- `QuickQuestions`：快捷问题按钮组。

### 5. 专业学习资料论坛模块

| 项目     | 说明 |
|----------|------|
| 功能说明 | 话题发布、评论（含二级）、文件上传下载、搜索与排序。 |
| 需求依据 | 需求文档 2.3 专业学习资料论坛 FR-011～FR-014。 |
| 接口     | 技术文档 3.4：话题 CRUD、评论、文件等。 |

**5.1 话题列表页（Forum / TopicList）**

- **路由**：`/forum` 或 `/forum/topics`。
- **页面要素**：
  - 搜索框：按标题/内容/标签搜索（FR-014）。
  - 筛选：按专业、排序（时间、热度等）。
  - 列表：话题标题、摘要、发布者、专业、标签、浏览/点赞/评论数、时间。
  - 「发帖」按钮，跳转创建话题页。
- **接口**：`GET /api/forum/topics?page=1&page_size=20&major_id= &keyword= `。

**5.2 话题详情页（Forum / TopicDetail）**

- **路由**：`/forum/topics/:id`。
- **页面要素**：
  - 标题、正文、发布者、专业、标签、时间、浏览/点赞/评论数。
  - 附件列表：文件名、大小、下载（FR-013）。
  - 评论列表：一级评论 + 二级回复（FR-012）；点赞。
  - 操作：评论输入框 + 提交；登录用户可点赞、收藏（若接口支持）。
- **接口**：话题详情、评论列表、发表评论、文件下载等。

**5.3 创建话题页（Forum / CreateTopic）**

- **路由**：`/forum/topics/create`。
- **页面要素**：
  - 标题、内容（富文本或纯文本）、专业选择、标签（FR-011）。
  - 文件上传：多文件，类型/大小校验（FR-013）；展示上传列表与删除。
  - 「发布」调用 `POST /api/forum/topics`（multipart/form-data）。
- **组件**：复用或封装 `FileUpload` 组件。

### 6. 课表查询模块（备注模式）

| 项目     | 说明 |
|----------|------|
| 功能说明 | 按周展示课表，支持学期切换、备注的增删改、按课程/教师/地点搜索。 |
| 需求依据 | 需求文档 2.5 课表查询 FR-019～FR-022。 |
| 接口     | `GET /api/schedule`、`PUT /api/schedule/{schedule_id}/note`（技术文档 3.5）。 |

**6.1 课表查询页（Schedule）**

- **路由**：`/schedule`。
- **页面要素**：
  1. **学期与筛选**
     - 学期选择（FR-019）；可选搜索：课程名、教师、地点（FR-021）。
  2. **课表主体**
     - 周视图：周一～周日 × 节次网格；单元格内展示课程名、时间、地点、教师（FR-019）。
     - 点击某节课可展开/弹窗：查看备注、编辑备注、删除备注（FR-020）。
  3. **备注**
     - 单条备注的展示与编辑（作业、考试、课程重点等）；保存调用 `PUT /api/schedule/{schedule_id}/note`。
- **交互**：周切换、loading、空状态提示；窄屏可考虑左右滑动切换周。

**6.2 组件建议**

- `ScheduleGrid`：周课表网格。
- `CourseCell`：单格课程信息 + 备注入口。

### 7. 时间规划表模块（日历模式）

| 项目     | 说明 |
|----------|------|
| 功能说明 | 月/周/日日历视图，事件增删改，类型与提醒，与课表同步。 |
| 需求依据 | 需求文档 2.4 时间规划表 FR-015～FR-018。 |
| 接口     | `GET /api/time-plan`、`POST /api/time-plan/events` 等（技术文档 3.6）。 |

**7.1 时间规划页（TimePlan）**

- **路由**：`/time-plan`。
- **页面要素**：
  1. **视图切换**
     - 月视图、周视图、日视图（FR-015）；当前日期与今日高亮。
  2. **日历主体**
     - 按视图展示事件；支持按类型区分样式（课程/考试/作业/个人等）（FR-016）。
     - 点击某日或某格可新建事件；点击已有事件可编辑/删除。
  3. **事件编辑**
     - 标题、类型、开始/结束时间、地点、描述、提醒时间（FR-016）。

  4. **课表同步**
     - 若有「从课表同步」入口，调用后端同步接口，在日历中展示课程事件（FR-017）。
- **交互**：今日/当前时间线高亮；窄屏可滑动切换日期；提醒可依赖后端或本地通知（按技术方案）。

**7.2 组件建议**

- `Calendar`：月/周/日切换的日历容器（推荐 FullCalendar Vue 组件，或 Element Plus + 自研视图）。
- `EventModal`：事件新增/编辑弹窗。
- `EventChip`：单条事件在日历上的展示块。

## 五、通用约定

1. **接口**：所有请求通过封装的 Axios 实例（及 `api/modules` 拆分）统一发送，携带 Token；错误统一处理（如 401 跳转登录）。后端未就绪时可在模块内切换 mock，与毕设阶段规划一致。
2. **权限**：需登录的路由在 **Vue Router 导航守卫**（`beforeEach` 等）中校验登录态；未登录则跳转登录。
3. **编码与注释**：UTF-8，中文注释；组件与页面需简要注释用途。
4. **兼容性**：支持 Chrome、Firefox、Safari、Edge；布局响应式，兼顾桌面与移动浏览器（对应需求文档 3.4 / NFR-007 的 Web 侧落实）。
5. **错误与加载**：列表/查询页有 loading；接口失败有明确提示，避免白屏。

## 七、给 Studio 的分阶段生成 Prompt

本节用于把前端页面交给 AI Studio、Bolt、StackBlitz 等工具先生成原型代码，再下载或复制到本地 `frontend/` 工程中继续修改。建议每次只生成一个阶段，避免一次性生成过多文件导致结构混乱。

### 使用原则

1. **技术栈必须写死**：Vue 3、Vite、Vue Router、Pinia、Element Plus、Axios。
2. **只做 Web 端**：不要生成小程序、React、Next.js、后端代码。
3. **先 mock 后接口**：页面先用本地 mock 数据跑通，接口函数要保留，后续可替换为 FastAPI。
4. **结构要清晰**：页面放 `src/views/`，组件放 `src/components/`，接口放 `src/api/`，状态放 `src/stores/`。
5. **中文场景**：页面文案、变量注释、提示信息使用简体中文，文件统一 UTF-8。
6. **不要过度设计**：先满足毕设演示，避免引入复杂权限、多租户、复杂主题系统等暂时用不到的内容。
